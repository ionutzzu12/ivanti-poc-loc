カタログ内のどのアプリについても、アプリカタログからアプリ詳細に進むことができます。 アプリ詳細のページには、ディスプレイバージョン（例：1.5.0）、バンドルバージョン（例：1.5.0.42）、最小OSバージョン（例：Android 5.0）など、アプリの詳細情報が表示されます。

\[最小OSバージョン] フィールドに指定されたバージョン以外のアプリはApps\@Workカタログに表示されません。 したがってそのようなアプリはデバイスに配布できません。 \[最小OSバージョン] フィールドはアプリの[**監査証跡**](./ダッシュボード.md)の一部としても表示されます。

**手順**

1. **\[アプリ]** をクリックします。
2. **\[アプリカタログ]** をクリックします。
3. アプリを選択します。

\[アプリの詳細] ウィンドウが表示されます。

iOSの自社開発アプリの場合、アプリ詳細ページで **\[プロビジョニングプロファイル有効期限]** を確認できます。

アプリ情報に、すべてのiOSおよびiPadOS VPPアプリのオプションとして、**\[アプリ配布時にM1デバイスへのインストールを許可]** が表示されるようになりました。 管理者は、M1 macOSデバイスにインストールできるiOSまたはiPad VPPアプリケーションに対してのみ、**\[アプリ配布時にM1デバイスへのインストールを許可]** オプションを有効化する必要があります。 このオプションを有効化した場合のみ、アプリケーションのインストール中に管理者に対してM1 macOSデバイスが表示されます。 マネージドアプリ構成は、M1 Mac デバイスの iOS VPP アプリケーションでサポートされます。

iOSのAppStoreアプリの場合、**\[新しいバージョンを同期]** を実行すると、iTunes Store内のアプリ更新がチェックされ、同期されます。 更新後のアプリバージョンは、アプリ同期スケジュールに基づいて12～24時間以内に表示されます。

\[デバイスの詳細] に **\[必須アプリ]** のトグルボタンが追加されました。 管理者は、このオプションを選択し、メインのアプリの前提条件となるアプリを追加することができます。

アプリを各種デバイスに配布した後は、アプリの配布ステータスを **\[デバイスの概要]** タブで確認できます。 また、\[デバイスの概要] タブには、資格のあるデバイスの数、アプリがインストールされたデバイスの数、アプリのインストールが保留になっているデバイスの数、アプリのインストールが失敗したデバイスの数も表示されます。 これらのボックスに表示されるデバイス数は、アプリのインストールを要求した（したがって必須アプリに関して管理者が初期配布を追跡できるようになっている）デバイスに基づいています。

（ユーザーがインストールしたアプリケーションを含む） 全般的なアプリケーション数については、\[アプリのインベントリ] 検索を使用してください。

デバイスから除外されたアプリは、\[保留中] という配布状態になります。

iOS / OSXアプリに関する、アプリインストールの失敗理由を、次の表に示します。

| 理由                         | 説明                                                                           |
| -------------------------- | ---------------------------------------------------------------------------- |
| AppAlreadyInstalled        | 「アプリは既にインストール済みです。」                                                          |
| AppStoreDisabled           | 「アプリストアが無効になっています。」                                                          |
| CouldNotVerifyAppID        | 「App IDを検証できませんでした。」                                                         |
| Failed                     | 「不明なエラーによりアプリのインストールが失敗しました。」                                                |
| ManagedButUninstalled      | 「アプリは管理対象ですが、ユーザーによって削除されています。 アプリを再度インストールすると（ユーザーによる場合でも）、再度管理されるようになります。」 |
| ManagementRejected         | 「管理対象外アプリを管理対象アプリに変換することをユーザーが拒否しました。」                                       |
| NotAnApp                   | 「アプリケーション識別子が無効であり、アプリ以外の資産に解決されたため、InstallApplicationコマンドが拒否されました。」         |
| NotSupported               | 「アプリはサポートされていません。」                                                           |
| Other                      | 「新たに追加されたエラー理由です。」                                                           |
| PurchaseMethodNotSupported | 「iOS 7以降の場合は、購入方法が無効であるため、アプリをインストールできません（VPPアプリのみに適用されます）。」                 |
| UserInstalledApp           | 「管理対象アプリのインストールを実行できるようになる前に、ユーザーがアプリをインストールしました。」                           |
| UserRejected               | 「アプリをインストールするという提案をユーザーが拒否しました。」                                             |
| UpdateRejected             | 「アプリを更新するという提案をユーザーが拒否しました。」                                                 |

| 理由                      | 説明                                                                                  |
| ----------------------- | ----------------------------------------------------------------------------------- |
| USER\_INSTALLED         | 「既にデバイス（MDMの外部）にインストールされています。」                                                      |

| UNSUPPORTED\_INSTALL    | 「3.0 Androidクライアント/22.0.0サーバーに適合するように更新されました：インストール対アンインストールを追跡するための特定のステータスが必要です。」 |

| OTHER                   | 「新たに追加されたエラー理由です。」                                                                  |

Windowsデバイスの場合、\[アプリ構成] でアプリがサイレントインストールに設定されると、\[デバイスの概要] には、そのアプリについて、資格がある、インストール済み、保留中、または失敗したデバイスの数が表示されます。

## アプリ詳細ページからアプリ インベントリ情報を表示する

アプリ インベントリ情報を表示するには、**\[アプリ インベントリの表示 (すべてのバージョン)]** をクリックすると、**\[デバイス] > \[アプリ インベントリ]** にそのアプリのバンドル ID でフィルタリングされた一覧が表示されます。
