## 現行リリース - R118

### Appleパスキー認証

特定のパスキーに対してWebAuthnエンタープライズ認証を許可するようにデバイスを構成するには、パスキー認証構成を使用します。

管理対象デバイス上にパスキーが作成され、SecurityPasskeyAttestationはエンタープライズマネージド認証情報を使用して会社の認証情報を含むパスキーを認証します。 サポートされる認証情報は、SCEP、ACME、IDなどの認証情報です。

パスキー認証は以下をサポートしています。

- サポートされている最小オペレーティングシステムバージョン：macOS 14
- サポートされている登録方法：デバイス登録
- パスキーの同期を制御するには、管理対象Appleアカウントが必要です。
- ABMのアクセス管理と併用できます。

パスキーのペイロードはMacでは失敗します。 証明書資産は、認証済みエンドポイントではホストされません。

パスキー構成を作成する際は、最初にID証明書構成を作成し、それをパスキー構成にリンクする必要があります。 また、セキュリティパスキー認証構成セクションでは、以下の詳細を入力してください。

| **フィールド**              | **説明**                                                             |
| ---------------------- | ------------------------------------------------------------------ |
| 名前                     | パスキーの名前。                                                           |
| 説明                     | 構成に関する任意の情報。                                                       |
| 認証ID資産参照\* 認証IDキーは抽出可能 | パスキー構成にリンクするために作成した資産参照IDを入力します。IDキーが抽出可能である場合は、このオプションを選択して確定します。 |
| 依存する関係者                | 認証済みドメインの詳細を入力します。                                                 |

### Apple制約構成：

**制約設定の強化**：**\[制約設定]** ページのユーザインターフェイスでは、**Appleの制約構成**の外観が一新され、次の変更が行われています。

- 以前、管理者はiOSとmacOSの制約を別々に構成する必要がありました。 現在、管理者は **\[構成]** > **\[+追加]** に移動し、**\[Appleの制約]** を選択できます。 さらに管理者は、ドロップダウンリストから **\[プラットフォームのタイプ]** を選択し、新しいApple制約を追加できるようになりました。
- **\[制約設定]** ペインでは、複数の設定が **\[デバイス機能]**、**\[アプリケーション]**、**\[セキュリティおよびプライバシー]** など、関連性のあるタブの下に分離されました。 Appleが新しい制約をリリースした場合、その新しい制約は自動的にそれぞれのタブに追加されます。
- **\[制約設定]** ペインで管理者は、**\[デバイスにプッシュ]** トグルボタンを使用して、選択した設定をデバイスにプッシュできるようになりました。
